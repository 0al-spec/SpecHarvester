# Package Relation Proposals

Status: Producer review evidence

`package-relation-proposals.json` records deterministic relation proposals for
package-set candidate bundles generated by `draft-package-set`.

The artifact is producer-observed review material. It does not accept relations,
mutate SpecPM sources, publish registry metadata, or imply trust inheritance
between package-set members.

## JSON Identity

```json
{
  "apiVersion": "spec-harvester.package-relation-proposals/v0",
  "kind": "SpecHarvesterPackageRelationProposals",
  "schemaVersion": 1
}
```

The artifact is written beside `package-set-draft.json`:

```text
candidates/xyflow-package-set/package-set-draft.json
candidates/xyflow-package-set/package-relation-proposals.json
```

## Initial Relations

P25-T4 starts with `contains` relation proposals from aggregate workspace
packages to selected scoped member packages.

For the `xyflow` reference path this produces:

```text
xyflow.workspace contains xyflow.system
xyflow.workspace contains xyflow.react
xyflow.workspace contains xyflow.svelte
```

Each relation records:

- relation `type`;
- `source.packageId`;
- `target.packageId`;
- source and target candidate paths;
- `reviewStatus: producer_observed`;
- workspace manifest evidence paths;
- source and target package manifest evidence paths;
- digest records when available from `workspace-inventory.json`.

## Input References

The relation proposal artifact references:

- `workspace-inventory.json`, including digest, `apiVersion`, and `kind`;
- `package-set-draft.json`, including digest, `apiVersion`, and `kind`.

The relation artifact does not hash itself. `preflight-bundle-set` verifies
relation source/target existence and checks the relation input digest for the
current `package-set-draft.json`.

## Boundary

Relation proposals are not SpecPM accepted registry metadata. Maintainers can
accept, reject, or defer each relation independently in the SpecPM review path.

SpecHarvester does not:

- accept package relations;
- publish public registry metadata;
- infer inherited capabilities or constraints;
- execute package scripts;
- install dependencies;
- run package managers.

`preflight-bundle-set` owns bundle-set preflight across candidate bundles and
relation output. P25-T6 owns static viewer relation badges and package-set
review panels.
